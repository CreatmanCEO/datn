# AI-агенты DATN

## Общие принципы

Каждый агент в DATN:
- **Независим** — не знает о существовании других агентов
- **Стандартизирован** — реализует `BaseAgent`, возвращает `AgentSignal`
- **Оценивается** — имеет метрику исторической точности, влияющую на его вес в консенсусе
- **Изолирован** — падение одного агента не влияет на работу остальных

---

## 1. Sentiment Agent (FinBERT)

### Назначение
Анализирует тональность финансовых новостей и постов в соцсетях для определения рыночного настроения.

### Модель
**ProsusAI/finbert** — BERT, дообученный на финансовых текстах. Три класса: positive / negative / neutral.

### Pipeline
1. Получает последние N новостей/постов по символу
2. Каждый текст проходит через FinBERT
3. Агрегация: взвешенное среднее с учётом свежести (экспоненциальное затухание), источника (Reuters > случайный блог), длины текста
4. Формирование сигнала

### Характеристики
- **Latency:** ~200ms на текст (batch inference быстрее)
- **Сильные стороны:** Хорошо ловит panic/euphoria, реагирует на крупные события
- **Слабости:** Запаздывает относительно цены, шум от мелких новостей
- **Лучшая точность:** На новостях о регулировании, крупных сделках, макроэкономике

## 2. Forecast Agent (Chronos-T5)

### Назначение
Прогнозирование будущих цен на основе исторических временных рядов.

### Модель
**Amazon Chronos-T5** — foundation model для временных рядов на базе T5. Генерирует вероятностные прогнозы (не точечные).

### Pipeline
1. Берёт последние 512 свечей
2. Генерирует 100 сценариев (Monte Carlo sampling)
3. Анализирует распределение: медиана, квантили 10/90
4. Direction из изменения медианы, confidence из ширины распределения (узкое = уверен)

### Характеристики
- **Latency:** ~500ms (GPU), ~2s (CPU)
- **Сильные стороны:** Хорош на trending рынках, даёт uncertainty estimation
- **Слабости:** Плохо работает при резких сменах режима, не учитывает фундаментал
- **Лучшая точность:** Среднесрочные тренды (4h-1d таймфрейм)

## 3. Anomaly Agent (Isolation Forest + Z-score)

### Назначение
Детекция аномальных паттернов в рыночных данных: необычные объёмы, спреды, резкие движения.

### Pipeline
1. Feature engineering: объём, волатильность, спред, скорость изменения цены
2. **Isolation Forest** — многомерная детекция аномалий
3. **Z-score** — простые статистические выбросы по объёму и волатильности
4. Направление: аномально высокий объём при росте = bullish

### Характеристики
- **Latency:** ~50ms
- **Сильные стороны:** Раннее предупреждение, ловит pump/dump, инсайдерскую торговлю
- **Слабости:** Не определяет направление сам по себе, ложные срабатывания
- **Лучшая точность:** Криптовалюты, акции с низкой ликвидностью

## 4. Technical Agent (TA-Lib)

### Назначение
Классический технический анализ: тренды, уровни, осцилляторы.

### Pipeline
Стратегия голосования: каждый индикатор "голосует" за направление, результат — взвешенная сумма.

**Индикаторы:**
- Trend: SMA 20/50 кроссовер (0.15), EMA 12/26 (0.15), MACD signal line (0.20), ADX (0.10)
- Momentum: RSI overbought/oversold (0.15), Stochastic %K/%D (0.10), CCI (0.05)
- Volume: OBV (0.05), VWAP (0.05)

### Характеристики
- **Latency:** ~30ms
- **Сильные стороны:** Быстрый, стабильный, работает на любых таймфреймах
- **Слабости:** Запаздывающие индикаторы, плохо на боковом рынке
- **Лучшая точность:** Trending рынки, крупные таймфреймы (4h+)

## Добавление нового агента

1. Создать файл в `src/agents/`
2. Наследоваться от `BaseAgent`
3. Реализовать `analyze()` и `health_check()`
4. Зарегистрировать в `Orchestrator`
5. Агент автоматически получит начальный вес 0.5 в консенсусе

```python
class MyNewAgent(BaseAgent):
    async def analyze(self, symbol: str, data: dict) -> AgentSignal:
        # Ваша логика
        return AgentSignal(...)
    
    async def health_check(self) -> bool:
        return True
```
